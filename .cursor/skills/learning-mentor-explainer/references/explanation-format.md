# Format wyjaśnień (mentor mode)

## 1. Założenia
- Co zakładamy o kontekście użytkownika / projektu.
- Jakie pojęcia już występują w repo / roadmapie.

## 2. Wyjaśnienie
### 2.1 Background (po co mechanizm istnieje)
- Jaki problem rozwiązuje.
- Co by się stało bez niego.

### 2.2 Jak działa pod maską
- Co robimy → Po co → Jak poznać, że działa
- Krok 1:
- Krok 2:
- Krok 3:

## 3. Jak poznam, że działa (operacyjnie)
- Jakie sygnały / logi / statusy / efekty.
- Jakie smoke testy.

## 4. Pułapki / antywzorce
- Typowe błędne założenia
- Typowe objawy błędów

## 5. Dalej
- Co doczytać / co przetestować w projekcie
- Jaki kolejny task tygodnia ma sens
